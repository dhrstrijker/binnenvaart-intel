Root cause: one model for many markets

Where the model works

Modern (2000+), large (110m+), Motorvrachtschip
MAPE ~19% | Bias ~-6% | Well-calibrated

This is a predictable commodity market: price is driven by length, capacity, and age. The linear model captures this well.

The sweet spot

ConditionMAPE
110m+ vessels18.5%
Built 2010s19.0%
Built 2000s20.9%
Galle source19.2%

Where it catastrophically fails

5 failure modes compound into 51 broken predictions:
1. Heritage vessels
Pre-1950 vessels have conversion/heritage value the model can't see. 51 produce negative raw predictions.
2. Non-cargo types
Tugboats, houseboats, crane ships: priced by engine power, accommodation, equipment - not length/tonnage.
3. Small vessels (<60m)
71 vessels where the model extrapolates far outside its training range.
4. Missing features
No tonnage data = €823k error on tankers. Condition, renovation, certifications are invisible.
5. Linear assumption
Price-vs-age is non-linear (U-shaped for heritage). Price-vs-length has diminishing returns above 110m.

Actionable takeaways

Ordered by expected impact. These changes could cut overall MAPE from 51% to ~25%.

  • Suppress predictions for non-cargo types high impact

    Show "Geen schatting beschikbaar" for Duw/Sleepboot, Woonschip, Kraanschip, Accomodatieschip, Ponton, and Overige. These 30 vessels produce meaningless predictions and erode user trust. Don't show a deal score badge for them either.

  • Add a confidence gate: hide predictions below threshold high impact

    If build_year < 1960 or length_m < 50, show "Indicatief" warning or suppress entirely. This would prevent 51 floor-clamped predictions from showing misleading €10k estimates for vessels worth €50k-€700k.

  • Switch to log-price regression high impact

    Prices range from €50k to €15M (300x). A linear model on raw EUR predicts negative prices for small/old vessels. Log-transforming the target prevents negative predictions entirely and handles the heteroscedasticity (bigger vessels have bigger residuals).

  • Build a dedicated Duw/Sleepboot model medium impact

    20 vessels, 96% MAPE. Extract engine power (HP/kW) from raw_details and use it as the primary feature. Tugboat value is driven by propulsion, not by cargo capacity. Even a simple HP-based model would dramatically improve these predictions.

  • Extract condition/renovation signals from raw_details medium impact

    Old vessels with high prices have been renovated. The "Motorvrachtschip, 80m, 1955" pattern appears in both €155k and €595k price points. The difference is interior condition, which is buried in raw_details but never used as a feature.

  • Use Koppelverband-specific model with combined dimensions low impact

    17 vessels, 56% MAPE. Combination vessels (tug + barge) have different economics. Consider using combined length and total tonnage across units, or simply flagging these as multi-unit.

Bottom line

The model works well for its core segment: modern, large cargo vessels (MAPE ~19%). But it's being applied to vessel types it was never designed for. The quickest wins come from knowing when NOT to show a prediction rather than trying to make one model fit everything.
